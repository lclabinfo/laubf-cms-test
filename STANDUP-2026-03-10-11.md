# Standup Summary

## Infrastructure & Deployment

- **R2 Media Migration** -- Moved all 93 website assets (images, videos, logos) from `/public/` to Cloudflare R2 CDN. Seeded MediaAsset + MediaFolder records so the CMS media library reflects R2 state.
- **Deploy Scripts** -- Created 6 idempotent deploy-data scripts (`apply-church-profile`, `apply-reference-data`, `apply-people-data`, `apply-recurring-events`, `apply-website-data`, `verify-deploy`) plus a `deploy-all` orchestrator for server data deployment without touching content tables.
- **Domain & Proxy** -- Configured nginx proxy, `.env.example`, deploy scripts, and admin subdomain routing (`admin.laubf.lclab.io` for CMS, `laubf.lclab.io` for public website).
- **Church Profile Cleanup** -- Fixed address (Anaheim to Downey), corrected contact info, social links, YouTube channel IDs, trimmed fake events down to 4 real recurring meetings.

## Authentication & Access Control

- **Custom Roles System** -- Replaced hardcoded 4-role enum with flexible custom roles supporting 49 granular permissions across 17 groups. Role model with String[] permissions, priority hierarchy, `isSystem` flag.
- **Auth Flows** -- Signup, login, forgot-password, reset-password, verify-email, accept-invite API routes and CMS pages. Login rate limiting (5/15min).
- **User Management** -- Soft-delete (ACTIVE/INACTIVE/PENDING status), deactivate/reactivate, person-user auto-link on invite, role dropdown with descriptions.
- **API Security** -- `requireApiAuth()` with Permission strings on 56 handlers across 15 route files. Hardened session cookies, RoleGuard component.
- **Centralized Onboarding** -- `/cms/onboarding` for PENDING members. Atomic activation endpoint. JWT memberStatus re-check. Dashboard redirects by status.

## Content Pipeline

- **Bible Study Re-migration** -- Full re-migration of 1,180 studies: 2,720 files converted (1,204 .docx, 1,436 .doc, 80 .rtf). Includes serif font detection, paragraph font wrapping, list font-family, hanging indent with tab-size, ordered list continuation.
- **TipTap Editor Improvements** -- Styled ordered/bullet lists with font propagation, `getParseExtensions()` for static conversion, fullscreen mode, sliding-indicator tabs for study sections.
- **DOCX/DOC Import Fixes** -- Tab handling, hanging indent, serif font wrapping, infinite loop fix in list item wrapping, WordPerfect 5.x detection and graceful skip.
- **Attachment System** -- Duplicate attachment dialog (Cancel / Upload Separately / Update Existing), restored attachment list on Details and Bible Study tabs.

## CMS Admin UI

- **Dashboard Redesign** -- Merged Content Health + At a Glance into StatsOverview with color-coded icons. Quick actions with gradients. Two-column layout. WCAG AAA font sizes.
- **Form Submissions Inbox** -- Full persistence pipeline (DAL, API, DB). Email-inbox UI with Sheet preview. Status workflow (new/reviewed/contacted/archived). Activity log. Batch operations. Notification email recipients configurable in CMS settings. Rate limiting, honeypot, validation hardening.
- **Storage Deep-Links** -- Largest Files table links media to the media library preview dialog (`?assetId=`) and attachments to the message editor's attachments section (`?section=attachments`).
- **Media Library** -- Video dialog, folder rename, redesigned MediaPickerDialog matching Figma, "Used in" section with scroll + modal, unsaved changes warning, R2 real file sizes, orphan cleanup script.
- **Roles Management** -- Card grid with create/edit/duplicate/delete. Permission picker with grouped checkboxes. Delete dialog with fallback role reassignment. Priority-based privilege escalation prevention.
- **Account Settings** -- Editable name on settings page. Session refresh from DB for immediate profile edits.
- **Tutorials** -- Interactive guided tutorials for new CMS users with spotlight targeting.
