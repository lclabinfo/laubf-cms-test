# 01. 비즈니스 요구사항

## Business Requirements Document

---

## 1. 비전 및 미션

### 1.1 비전
> 모든 규모의 교회에 접근 가능한 가격으로 엔터프라이즈급 디지털 도구를 제공하여, 교회가 온라인 영역을 확장하고, 성도 참여를 심화하며, 운영을 효율화할 수 있도록 돕는다.

### 1.2 미션
- 교회 특화 디지털 플랫폼 제공
- 복잡한 기술을 단순하게
- 비용 효율적인 솔루션
- 확장 가능한 인프라

### 1.3 핵심 가치
1. **교회 중심**: 교회 워크플로우와 사역에 최적화
2. **접근성**: 모든 규모의 교회가 사용 가능한 가격
3. **유연성**: 필요한 기능만 선택하여 사용
4. **보안**: 성도 데이터 보호 최우선
5. **혁신**: AI 기반 콘텐츠 생성으로 시간 절약

## 2. 시장 분석

### 2.1 타겟 시장

| 세그먼트 | 규모 | 특성 | 니즈 |
|---------|------|------|------|
| **소형 교회** | 50-200명 | 제한된 예산, IT 인력 부족 | 저렴하고 쉬운 솔루션 |
| **중형 교회** | 200-1,000명 | 성장 중, 다양한 사역 | 확장 가능한 통합 플랫폼 |
| **대형 교회** | 1,000-5,000명 | 전문 스태프, 복잡한 조직 | 고급 기능, 커스터마이징 |
| **메가 교회** | 5,000명+ | 다중 캠퍼스, 대규모 운영 | 엔터프라이즈급, 화이트라벨 |

### 2.2 시장 규모

```
전 세계 교회 관리 소프트웨어 시장:
├── 2024년: $1.5B
├── 2028년 예상: $2.8B
└── CAGR: 17.2%

미국 교회 수: ~380,000개
├── 온라인 헌금 사용 교회: 약 45%
├── 교회 관리 소프트웨어 사용: 약 35%
└── 잠재 시장: $500M+
```

## 3. 수익 모델

### 3.1 수익 구조

```
┌─────────────────────────────────────────────────────────────────┐
│                        수익 스트림                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  1. 구독 수익 (Recurring Revenue)                                │
│     ├── Plan 기반 월/연 구독                                     │
│     ├── 모듈별 구독                                              │
│     └── 목표: 총 수익의 60%                                      │
│                                                                  │
│  2. 거래 수수료 (Transaction Fees)                               │
│     ├── 헌금 처리 수수료 (2.0% + $0.25)                          │
│     ├── 플랫폼 수수료 (Stripe 마진)                              │
│     └── 목표: 총 수익의 30%                                      │
│                                                                  │
│  3. 부가 서비스 (Add-ons)                                        │
│     ├── 추가 스토리지                                            │
│     ├── 프리미엄 지원                                            │
│     ├── 화이트라벨                                               │
│     └── 목표: 총 수익의 10%                                      │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 Unit Economics

```
예상 고객 당 수익 (Monthly):
├── 평균 모듈 구독: $120
├── 평균 헌금 수수료: $80 (월 $4,000 헌금 기준)
├── 부가 서비스: $20
└── 총 ARPU: $220/월

예상 비용:
├── 인프라 (AWS/Vercel): $15/tenant
├── 결제 처리 (Stripe): $60/tenant
├── 지원 비용: $10/tenant
└── 총 비용: $85/tenant

예상 마진: $135/tenant (61%)
```

## 8. 로드맵 요약

### Phase 1: MVP (3개월)
- 핵심 멀티테넌트 인프라
- 기본 웹사이트 빌더
- 헌금 시스템 (Stripe)
- 성도 관리 기본

### Phase 2: Growth (3개월)
- 모바일 앱 (React Native)
- 이벤트 관리
- 소그룹/사역
- 이메일 알림

### Phase 3: Premium (3개월)
- 라이브 스트리밍
- 어린이 체크인
- 고급 분석
- API 액세스

### Phase 4: AI & Scale (3개월)
- AI Shorts Creator
- AI Social Studio
- 엔터프라이즈 기능
- 글로벌 확장
